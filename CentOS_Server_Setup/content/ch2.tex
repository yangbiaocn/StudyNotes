\chapter{CentOS安装问题与解决方法}
\section{配置网络适配置器}
\begin{description}
  \item[问题] CentOS 7.0在Vmware上安装完成后，无法识别网卡，从而导致虚拟机无法上网。
  \item[解决方法] 找到Vmware虚拟机文件夹，将VMware 虚拟机配置 (.vmx)，追加一条设置我们网卡类型ethernet0.virtualDev = "e1000"
  \item[原因] Vmware虚拟网卡和linux兼容问题导致驱动无法正常安装，默认的网卡类型不兼容
\end{description}

VMware都在虚拟客户机中提供的是古老的AMD PCNet10/100Mbps虚拟网卡，后来，VMware通过一种类似部分虚拟化的vmxnet优化虚拟网卡的方式提供了10/100 /1000Mbps性能，无疑性能更好。现在，VMware ESX Server已经可以提供基于Intel E1000的千兆虚拟网卡，从兼容性和性能的角度来看，E1000都是更好的选择。但似乎，在ESX/ESXi 3.5的图形界面中，都无法直接选择该网卡类型，需要手动配置。
类型可以选择多项
\begin{description}
\item[值为"e1000"]指定网卡类型为Intel(R) PRO/1000
\item[值为"vlance"]指定网卡类型为AMD PCNet AM79C970A（默认为此项不兼容）
\item[值为"vmxnet"]指定网卡类型为VMware PCI Ethernet Adapter
\end{description}
